route.request module

Route.Request module is in charge of parsing and handling incoming JSON requests.

type route.request.AuthFunc = Callable[[str], Status]

Authorisation functions take an authorisation header (contains token) and return a status.Status.

type route.request.CredFunc = Callable[[list[str]], Status]

Credential functions take a list of credentials and return a status.Status. For example, a Supabase login function will likely take “email” and “password” as credentials.

route.request.auth_call(func: AuthFunc) → Status

Reads the authorisation header, then runs the supplied function as the authorised user.

Parameters:

func – The function that will be called with the authorisation header as its sole argument.

Returns:

The jsonified response of calling the function, along with a status code.

route.request.cred_call(creds: list[str], func: CredFunc) → Status

Reads and passes the specified credentials to the supplied function.

Parameters:

  • creds – The credentials that need be extracted from the flask request (give them in the same order that they should be passed to the specified function).

  • func – The function that will be called with the read credentials passed to it.

Returns:

The jsonified response of calling the function with the credentials, along with a status code.
