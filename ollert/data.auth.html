data.auth module

Data.Auth module handles Supabase authorisation calls.

data.auth.parse_token(data)

Helper function used by sb_login() and sb_register() to extract essential token information from a large Supabase response.

Parameters:

data – Large unparsed Supabase response.

Returns:

Dictionary containing solely token information.

data.auth.sb_login(email: str, password: str) → Status

Uses email and password to attempt to login with Supabase.

Parameters:

  • email – Email of user.

  • password – Password associated with user.

Returns:

  • Success: Access and refresh tokens that the user can store.

  • Failure: [401] BAD_AUTH error code.

data.auth.sb_logout(auth_header: str) → Status

Uses access token within authorisation header to logout the associated user.

Parameters:

auth_header – Header read from HTTP request containing access token.

Returns:

  • Success: [200] SUCCESS code.

  • Failure: [401] BAD_AUTH error code.

data.auth.sb_register(email: str, password: str, full_name: str) → Status

Uses email, password and full name to attempt to register with Supabase.

Parameters:

  • email – Email of user.

  • password – Password to be set for user.

  • full_name – All names of user that they wish be stored.

Returns:

  • Success: Access and refresh tokens that the user can store.

  • Failure: [401] BAD_AUTH error code.
